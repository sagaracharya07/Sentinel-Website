Help & Documentation

Administrator-focused guidance. See also the public Help Centre and Threat Lab.

Connecting Gmail

Open Connected Mailboxes and click Connect Gmail. Grant the requested permissions in Google's consent screen, then confirm labels are ready with Test Connection.

Understanding verdicts

Legitimate / Needs Review / Phishing come from the classifier's phishing probability plus structured security findings — see Detection Policy in Settings for the current threshold values.

Reviewing incidents

Open any detection to see sender identity, authentication, links, attachments, and the model's explanation, then Confirm, Release, or Mark Legitimate.

Releasing a quarantined message

Release only reports success once Gmail confirms the label move — if it fails, the message stays quarantined and an error is shown.

Connection troubleshooting

Test Connection re-verifies credentials and labels. If it fails with an authentication error, Reconnect to refresh Google's consent and issue a new token.

Current limitations

One connected Gmail mailbox per deployment; no independent DNS validation of SPF/DKIM/DMARC beyond the provider's reported results; related-message grouping is basic (sender/domain).
