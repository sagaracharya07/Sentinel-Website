Features

Every part of the pipeline is built to be inspected, not just trusted.

Sentinel AI combines a trained machine-learning classifier with a set of hand-engineered signals that mirror what a human analyst looks for — so every verdict comes with a real explanation, not just a score.

Explainable scoring

A Random Forest model trained on TF-IDF text features is combined with engineered signals — urgency language, credential requests, generic greetings, shortened/raw-IP links, sender/brand mismatches, formatting anomalies — so every flagged email shows exactly which phrases and links drove the score.

Live mailbox monitoring

Connect a real Gmail, Outlook, or Yahoo inbox over IMAP. A background job polls for new mail, classifies it with the same model used everywhere else on the platform, and takes a real action automatically.

Reversible quarantine

High-risk mail is moved to a dedicated quarantine folder — never deleted. Medium-risk mail is flagged in place for review. Releasing a false positive moves the real message back to the inbox, looked up by its stable Message-ID.

Feedback-driven retraining

Users and admins can correct a verdict in one click. Confirmed corrections are folded into a new training run on demand, producing a new versioned model with its own measured accuracy, precision, and recall — hot-swapped in with no downtime.

Full audit trail

Every scan, admin action, login, retrain, and mailbox sync is timestamped and logged. Nothing about a decision's history is hidden from review.

Data minimisation by default

Raw email bodies are automatically redacted 24 hours after a scan, keeping only the classification metadata needed for auditing — a real background job, not a policy on paper.
