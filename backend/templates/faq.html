FAQ

Frequently asked questions

How accurate is the detection model?

On a held-out 20% test split, the current model measures roughly 96% accuracy, 95% precision, and 97% recall. These are real numbers from a real train/test split, reproducible by running the training script yourself — not a marketing figure.

Does Sentinel AI delete any email?

No. High-risk mail is moved to a quarantine folder (reversible at any time from any mail client); medium-risk mail is flagged in place. Nothing is ever permanently deleted by the platform.

What happens to the content of a scanned email?

The email body is stored only long enough to support review and feedback — a background job automatically redacts it 24 hours after the scan, keeping just the classification metadata.

Can I connect more than one mailbox?

Not in the current build — one deployment watches one mailbox, representing a shared organisational inbox. Multi-mailbox support is a planned extension.

What if the model gets something wrong?

Correct it in one click from the scan result or the admin console. Confirmed corrections are folded into the next retrain, which produces a new, independently measured model version.

Is my password ever visible to anyone, including admins?

No — passwords are hashed with PBKDF2 before storage and are never logged. Not even an administrator can retrieve a user's actual password.

Do I need to be technical to set up mailbox monitoring?

You need an IMAP host and an app password from your provider (Gmail, Outlook, or Yahoo) — the same credential type used by most third-party mail apps. See the Integrations page for provider-specific steps.
