How it works

One pipeline, every email, no exceptions.

Whether a message comes from a live monitored mailbox or is pasted manually into the scanner, it moves through the exact same steps — nothing about how mail is judged changes based on where it came from.

01

Ingest

A background job polls the connected mailbox over IMAP every ~45 seconds and pulls any message not already scanned, or a user pastes one in manually.

→
02

Feature extraction

The subject and body are turned into a TF-IDF vector plus a set of engineered signals — urgency phrases, credential requests, suspicious links, sender mismatch, formatting anomalies.

→
03

Classification

A trained Random Forest model scores the combined feature vector and produces a 0–100 confidence score and a Low/Medium/High risk band.

→
04

Action

High risk is quarantined into a dedicated folder. Medium risk is flagged in place. Low risk is delivered untouched — every action is logged.

→
05

Feedback & retrain

A user or admin can correct any verdict in one click. Confirmed corrections are folded into the next retrain, producing a new, independently-measured model version.

Connecting a mailbox

Sentinel AI watches one mailbox at a time — representing an organisation's protected inbox rather than every individual employee's account. Setup uses an IMAP host, username, and an app password (never your real account password), the same credential model Gmail, Outlook, and Yahoo all support for third-party mail access.

Once connected, an admin can see live connection status, trigger an immediate sync, and test the connection at any time from the admin console.

Why explanations matter more than a label

A Random Forest's TF-IDF half isn't individually interpretable — you can't point at one weight and say "this is why." So Sentinel AI's engineered signal half exists specifically to carry the explainability requirement: every finding shown to a user (a brand mismatch, an urgency phrase, a shortened link) is a concrete, human-checkable reason, not a black-box confidence number alone.

What retraining actually changes

Clicking "Retrain" doesn't tweak the live model in place — it trains an entirely new, independently versioned model against the full dataset plus every confirmed correction since the last training run, measures its accuracy/precision/recall/F1 on a held-out test split, and only then swaps it in. Every past version and its real metrics stay recorded, so the retraining history is auditable rather than just a claim.
