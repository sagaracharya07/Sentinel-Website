Product

Everything Sentinel does, organised around one workflow

Protect, Investigate, Respond, and Report — the four capabilities that make up the CONNECT → DETECT → INVESTIGATE → REMEDIATE lifecycle.

01 · Protect

Gmail monitoring that never sleeps

Connect a mailbox once with Google OAuth. Sentinel detects mailbox changes through the Gmail API, retrieves each new message, and classifies it into one of three states.

  • Continuous Gmail API monitoring (polling, with optional push via Pub/Sub)
  • Three-state verdicts: Legitimate, Needs Review, Phishing
  • Gmail labels applied automatically for each state
  • Phishing quarantined via label move — never deleted

02 · Investigate

See exactly why a message was flagged

Every detection opens into a full incident view with structured, explainable findings — never a single opaque score.

  • Sender identity: display-name, Reply-To, Return-Path mismatches
  • SPF / DKIM / DMARC authentication results
  • Link analysis: displayed text vs. real destination
  • Attachment metadata (filename, type, size, hash — never executed)

03 · Respond

Take action, with a record of every decision

Confirm phishing, release a false positive back to the inbox, mark legitimate, or find related messages by sender or domain.

  • One-click confirm, release, or mark-legitimate
  • Release only succeeds after Gmail confirms the label move
  • Related-message search by sender or domain
  • Every action written to the audit log

Illustrative — the real actions live in the Administrator Console's Incident Detail page.

04 · Report

A reporting path for everyone

Any employee can upload a suspicious message's original .eml file for the same full analysis Gmail mail receives.

  • Upload the original .eml for full-fidelity analysis
  • Track status from submission to final verdict
  • Administrator review sets the final decision
  • Quick Analysis for a faster, more limited pasted-text check
