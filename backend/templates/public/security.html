Security & Privacy

How Sentinel handles your Gmail data

An honest account of permissions, storage, and limitations — as a final-year computing project, not a certified enterprise product.

Gmail permissions

Connecting a mailbox uses Google OAuth. Sentinel requests only the scopes needed to read messages, apply labels, and move messages between labels (for quarantine and release) — it does not request permission to send email as you or manage other Google services.

OAuth token handling

Sentinel stores an encrypted OAuth refresh token — never your Google password. Tokens are encrypted at rest using a key that is separate from the application's own secret key, and are never displayed in the interface, returned by any API, or written to logs.

What is stored

Classification metadata (verdict, risk level, findings) for every analysed message, and message content temporarily, to support investigation. Message bodies are purged after a retention window, retaining only the classification record.

What is not stored

Your Google password, OAuth access/refresh tokens in plain text, or attachment file contents (only metadata — filename, type, size, hash — is recorded; attachments are never opened or executed).

Quarantine behaviour

Quarantine and release are Gmail label operations. Sentinel never permanently deletes a message — "quarantine" moves it out of the inbox via a label, and "release" reverses that.

Administrator access

Only accounts with the Security Administrator role can connect, view, or manage a Gmail mailbox, or see organisation-wide detections. Employees see only their own submitted reports.

Audit logging

Every connection, disconnection, quarantine, release, and administrator decision is recorded in an audit log with actor, action, and timestamp — never message content or secrets.

Disconnecting Gmail

Disconnecting a mailbox immediately wipes its stored encrypted tokens from the database — this is a genuine revocation, not just a status flag.

Data retention

Message bodies are purged after a fixed retention window. Classification metadata, audit log entries, and reports are retained for the operation of the platform.

Prototype limitations

  • Sentinel is a final-year computing project, evaluated in a development/demo environment — not independently security-audited or commercially certified.
  • SPF/DKIM/DMARC results are read from the provider's own authentication headers; Sentinel does not perform independent DNS validation.
  • This deployment supports one connected Gmail mailbox at a time.
  • No compliance certifications (SOC 2, ISO 27001, or similar) are claimed.
