Integrations

Connects to the mailbox you already use.

Sentinel AI reads mail over standard IMAP, so setup takes an app password, not a migration. No plugin to install in the mail client itself.

G

Gmail

Turn on 2-Step Verification, generate an App Password under Google Account settings, and use imap.gmail.com as the host. Recommended for testing with a throwaway account first.

O

Outlook / Office 365

Use outlook.office365.com as the host. An app password is required the same way if two-factor authentication is enabled on the account.

Y

Yahoo Mail

Use imap.mail.yahoo.com as the host, with an app password generated from Yahoo's account security settings.

What Sentinel AI does with access

Read access is used to pull new messages for classification. The only write actions it ever performs are moving a high-risk message into a dedicated quarantine folder it creates itself, or flagging a medium-risk message in place — both fully reversible from any ordinary mail client, and both logged. It never deletes mail permanently and never sends mail on your behalf from that mailbox.

One mailbox, on purpose

The current build watches a single configured mailbox — representing an organisation's shared, protected inbox rather than every employee's personal account. Monitoring multiple mailboxes from one deployment is a planned extension, not something this build claims to do today.
