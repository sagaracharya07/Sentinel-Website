About

A phishing detector that shows its work.

Most spam filters give you a label and nothing else: flagged, or not. Sentinel AI started from a different premise — that a security tool a person is expected to trust should be able to explain itself in the same terms a human analyst would use: this link is shortened, this sender's domain doesn't match the brand it claims to be, this phrase is a known urgency pattern.

Where it started

Sentinel AI began as an NIT3004 IT Capstone Project at Victoria University — a proposal for an AI-based phishing detection platform, built out into a real, running system rather than a slide deck: a trained machine-learning classifier, a real database, a real IMAP mailbox integration, and a full retraining loop, not a mockup.

What "real" means here

Every capability described on this site is something you can find in the code and exercise yourself: the model's measured accuracy and precision come from an actual held-out test split, not a marketing number; the mailbox integration performs genuine, reversible IMAP actions; the audit log records what actually happened, not a curated summary.

Where it's honest about its limits

This is a small, focused platform, not an enterprise security suite. It monitors one mailbox at a time rather than every individual inbox in an organisation, polls for new mail on an interval rather than pushing instantly, and doesn't (yet) send outbound alerts beyond account/security email. Those are documented trade-offs, not hidden gaps.
